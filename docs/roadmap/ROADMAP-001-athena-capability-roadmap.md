# Purpose

定义 Athena 未来 3~5 年的能力演进路线，以"能力（Capability）"而非"功能（Feature）"作为研发主线。

---

# Development Operating Cycle

每一个 Capability 必须遵循以下生命周期：

Research
→ Architecture
→ Specification
→ Implementation
→ Verification
→ Knowledge Update

任何阶段不得跳过。

---

# Capability Maturity Driven Development

Athena 不采用固定时间驱动的研发模式，而采用能力成熟度驱动（Capability Maturity Driven Development，CMDD）。

研发节奏定义如下：

- Task：最小实现单元（小时级）
- Capability：完整业务能力（1~7 天）
- Sprint：一个可运行版本（1~2 周）
- Wave：一组相关能力成熟后形成的能力集
- Evolution：基于真实数据、真实交易、真实回测持续演进

## Maturity Levels

每个 Capability 必须经过以下成熟度等级：

Level 0 - Concept（概念验证）

Level 1 - Prototype（原型实现）

Level 2 - Working（可运行）

Level 3 - Verified（通过验证）

Level 4 - Production Ready（满足生产要求）

Level 5 - Self Improving（具备持续学习与优化能力）

任何 Capability 的升级均以证据（Evidence）、验证（Verification）和成熟度为依据，而不是以自然时间为依据。

Athena 的目标是持续提升能力成熟度，而不是完成预设时间计划。

# Phase Roadmap

## Phase 1 Foundation（Completed）

建立基础工程与治理体系。

## Phase 2 Decision Intelligence（Current）

建立完整投资决策链。

## Phase 3 Learning Intelligence

建立持续学习与策略优化能力。

## Phase 4 Autonomous Intelligence

建立多 Agent 协同能力，并始终保留 Founder 的最终决策权。

---

# Success Criteria

Athena 的研发目标不是不断增加功能，而是持续增强核心能力（Capability），使系统能够在真实市场中不断提升决策质量、风险控制能力和学习能力。
